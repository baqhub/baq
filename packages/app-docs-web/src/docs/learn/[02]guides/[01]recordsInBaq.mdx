# Records in BAQ

At its core, BAQ is a federated database for immutable pieces of data called records, exposed through a familiar HTTP API.

Within BAQ, everything is made of records. This includes core platform features like types and subscriptions. Let's see what a record is, how they're used to enable higher-level platform features, and how to design your own data model.
